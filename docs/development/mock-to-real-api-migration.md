# 从模拟数据到真实后端API的改造路线图

本文档提供了将前端模拟数据转换为真实后端API的系统性策略和步骤。

## 改造目标

将所有前端硬编码的模拟数据替换为真实的后端API调用，实现完整的前后端交互，使系统具备真实的数据持久化能力。

## 改造策略

### 第一步：回顾数据模型和API设计

1. **查看数据模型定义**
   - 回到项目根目录的共享包中查看统一定义的数据模型
   - 确保前后端使用一致的数据结构
   - 路径: `shared/models/`

2. **审查API接口定义**
   - 检查是否已经定义了API接口规范
   - 确认接口路径、参数和返回格式
   - 路径: `docs/api/`

### 第二步：后端准备工作

1. **实现数据库模型**
   - 根据共享的数据模型创建/更新数据库表结构
   - 添加必要的索引和约束
   - 确保数据库设计满足所有功能需求

2. **开发API端点**
   - 按照API规范实现各个端点
   - 实现必要的业务逻辑和数据验证
   - 添加认证和授权控制

### 第三步：数据库种子数据

1. **编写种子脚本**
   - 创建专门的数据库种子脚本
   - 包含基本的用户、视频、评论等测试数据
   - 路径: `backend/src/seeds/`

2. **使用测试数据生成工具**
   - 使用Faker.js等工具自动生成符合模式的假数据
   - 生成逼真的用户名、头像、内容等
   - 设定合理的数据量以测试系统性能

3. **导入现有模拟数据**
   - 将前端现有的模拟数据结构化后导入数据库
   - 保持开发过程中的视觉一致性
   - 确保关联数据的完整性

### 第四步：前端改造

1. **按功能模块逐步改造**
   - 按以下顺序进行模块改造:
     - 用户认证
     - 创作者数据和统计
     - 视频管理
     - 评论系统
     - 频道定制
     - 推荐系统
     - 订阅功能
     - 其他辅助功能

2. **创建API服务层**
   - 为每个功能模块创建对应的API服务
   - 路径: `frontend/src/services/api/`
   - 使用统一的错误处理和加载状态管理
   - 实现数据缓存策略

3. **更新组件**
   - 将组件中的模拟数据替换为API调用
   - 添加加载状态、错误处理和空状态UI
   - 优化用户体验，处理数据加载延迟

### 第五步：测试与调试

1. **单元测试**
   - 为API服务编写单元测试
   - 模拟API响应进行组件测试
   - 确保数据处理逻辑正确

2. **集成测试**
   - 测试前后端交互
   - 验证数据流和状态管理
   - 测试各种边缘情况

3. **端到端测试**
   - 模拟用户行为进行完整流程测试
   - 确保功能在真实环境中正常工作
   - 使用Cypress等工具自动化测试

## 最佳实践建议

1. **使用特性标志**
   - 使用特性标志(feature flags)控制新功能的发布
   - 在开发过程中可以轻松切换回模拟数据
   - 配置路径: `frontend/src/config/features.ts`

2. **增量改造**
   - 不要一次性替换所有模拟数据
   - 每完成一个模块就进行充分测试
   - 逐步将改动合并到主分支

3. **保留模拟数据作为备份**
   - 将模拟数据代码注释而非删除
   - 可用于前端独立开发和测试
   - 考虑将模拟数据重构为专门的服务

4. **使用环境变量**
   - 配置环境变量决定是使用模拟数据还是真实API
   - 便于不同环境下的开发和测试
   - 例如: `VITE_USE_MOCK_DATA=false`

## 注意事项

1. **保持沟通**
   - 前后端开发人员需要密切沟通
   - 共同制定API接口规范
   - 及时反馈和解决问题

2. **优先级管理**
   - 优先改造核心功能
   - 次要功能可以后期再完善
   - 保持项目整体可用性

3. **性能监控**
   - 监控API调用性能
   - 识别并优化慢查询
   - 实现必要的数据缓存

4. **回滚策略**
   - 每个改动都要有回滚计划
   - 保留关键点的快照
   - 确保出现问题时能快速恢复

## 常见问题解答

1. **如何处理API响应结构与现有模拟数据不一致的情况？**
   - 使用转换适配器统一处理数据格式
   - 考虑更新API设计以匹配现有结构
   - 在过渡期使用数据映射

2. **如何管理API密钥和认证凭据？**
   - 使用环境变量存储敏感信息
   - 不要在代码中硬编码密钥
   - 考虑使用密钥管理服务

3. **如何处理网络错误和异常情况？**
   - 实现统一的错误处理机制
   - 为用户提供有意义的错误消息
   - 记录详细错误日志便于调试

4. **如何保持前端在后端不可用时的可用性？**
   - 实现优雅的降级策略
   - 考虑使用离线模式
   - 实现请求重试机制
